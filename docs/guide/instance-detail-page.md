# 实例详情页

当前 `/local-instance/...` 页面已经作为实例详情页使用：

- 路径：`/local-instance/{gameId}/{challengeId}/{teamHash}`
- 主要用途：
  - 查看当前账号在该题目下的实例租约状态
  - 查看平台回填后的真实入口、`host:port` 和连接命令
  - 直接执行启动、续期、销毁、刷新和复制入口信息

## 适用场景

- 动态题把入口解析到平台内地址时，用来承接当前队伍的实例详情
- 本地联调或实例 provider 接入初期，用来确认入口分发和租约状态是否一致
- 不需要离开平台页面，就能继续核对续期策略和剩余时间

## 当前边界

- 仍然依赖当前登录账号和队伍报名状态
- 主要服务于平台内实例链路，不替代真实业务服务页
- 如果题目已经回填外部 `launch_url`，仍然建议以真实服务页面作为最终操作入口

## 页面约定

- 页面主体优先保留：
  - 当前租约状态
  - 实例入口
  - 租约策略
  - 操作范围
- 页面顶部现在还会直接带出当前比赛名、题目名和题目分类：
  - 从公开比赛页跳到实例详情页后，不需要再靠 `game_id / challenge_id` 自己对照当前上下文
- 页面里的“返回比赛页”现在会直接带回原比赛的对应题卡：
  - 会自动切回 `题目` 标签
  - 并滚动到当前实例所属题目的卡片位置
- 登录要求、状态读取失败、实例运行状态这三类页内结论，优先合并进“当前租约”里的紧凑摘要头
- 不再为同一租约状态额外堆多块 `UAlert`
- 如果当前账号还没登录、没有队伍、尚未报名、报名待审核，或比赛当前不开放实例：
  - 页面会直接给出对应的下一步入口，例如登录、队伍页或返回比赛页
  - 不再只把后端错误原样暴露成一条读取失败提示
- 销毁实例继续通过 `UModal` 确认，不直接在正文区执行危险操作
- 当实例已运行但尚未进入续期窗口时：
  - 页面会直接显示“距离开放续期还有多久”
  - 进入续期窗口后会自动静默刷新一次状态，避免必须手动刷新后才能看到续期入口

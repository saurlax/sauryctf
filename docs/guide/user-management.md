# 用户管理

`/console/admin/users` 提供最小可用的账号管理入口，仅对 `admin` 和 `super_admin` 开放。

当前能力：

- 查看全部用户
- 查看用户角色、状态、邮箱和创建时间
- 按关键词检索用户名、邮箱或用户 ID
- 按角色筛选
- 按状态筛选
- 按用户单独更新角色
- 按用户单独更新状态
- 使用表格化视图集中查看账号信息与行内调整入口
- 角色或状态变更提交前会进入一次确认弹层

当前约束：

- 当前登录账号不能把自己改成封禁
- 当前登录账号不能修改自己的角色
- 只有 `super_admin` 可以管理 `super_admin`
- 非 `super_admin` 不能把其他用户提升为 `super_admin`
- 前端会提前隐藏这类无效选项：
  - 当前账号自己的角色下拉里只保留当前角色
  - 当前账号自己的状态下拉里不会出现 `封禁`
  - 非 `super_admin` 的角色下拉里不会出现 `super_admin`

建议用法：

- 只在确有运维或权限调整需求时修改角色
- 封禁前先确认该账号是否仍承担比赛管理职责
- 默认管理员完成初始化后，应优先在“账号安全”页更新密码

当前展示约定：

- 用户名、邮箱、角色状态、创建时间和调整入口统一放在一张表格里
- 行内直接显示“当前账号”“有未保存修改”等状态，减少反复展开或跳转
- 保存前通过确认弹层再次展示目标用户、角色变更和状态变更
- 权限受限时，保留只读下拉与原因说明，不隐藏约束结果

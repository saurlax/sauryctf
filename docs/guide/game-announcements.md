# 比赛公告

当前比赛公告分成两层：

- `notice`
  - 适合放长期有效的比赛说明
  - 例如基础规则、报名要求、Writeup 约束
- 公告列表
  - 适合放赛时通知
  - 例如开赛提醒、实例维护、补充规则、截止时间提醒

## 当前能力

- 公开比赛页会读取 `GET /api/games/{id}/announcements`
- 控制台首页会汇总当前账号已关联比赛的最新公告
- 管理端支持：
  - 查看当前比赛公告列表
  - 发布新公告
  - 删除旧公告
- 公告按发布时间倒序展示，最新内容优先出现

## 当前设计

- 保留原有单条 `notice` 字段，避免把长期规则说明和赛时通知混在一起
- 公告列表保持极简，只记录：
  - 公告内容
  - 发布时间
  - 发布人 ID
- 先不做富文本、置顶、编辑、已读状态或实时推送

## 当前约束

- 只有公开且可访问的比赛，公开页才会返回公告列表
- 当前公告只支持纯文本内容
- 删除后不会保留历史版本

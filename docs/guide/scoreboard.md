# 排行榜

当前排行榜已具备两层信息：

- 队伍总榜
  - 排名
  - 队伍名
  - 总分
  - 解题数
  - 最后一次解题时间
- 分题统计
  - 题目名称
  - 分类
  - 当前分值
  - 解出队伍数
  - 一血 / 二血 / 三血队伍
  - 按分类分组展示
- 封榜信息
  - 当前是否已封榜
  - 封榜时间
- 分组信息
  - 当前筛选分组
  - 比赛可用分组列表

## 当前实现

- 接口：`GET /api/games/{id}/scoreboard`
- 返回：
  - `entries`：队伍总榜
  - `challenges`：分题统计
  - `is_frozen`：公开榜单是否已封榜
  - `freeze_time`：封榜时间
  - `division`：当前筛选的分组
  - `divisions`：比赛配置的分组列表
- 前端展示：
  - 排行榜标签页顶部现在会先展示一组摘要卡片：
    - 当前查看的是全部队伍还是某个分组
    - 当前公开排名里有多少队伍
    - 当前分题统计里有多少题
    - 当前是否已经封榜
  - 还会额外展示“当前榜单视图”说明，直接告诉选手自己看到的是实时公开榜单还是封榜后的冻结视图
  - 队伍总榜保留表格
  - `最后一次解题时间` 现在会直接使用真实提交时间，便于快速判断同分队伍的先后顺序
  - 如果比赛启用了分组，可切换“全部分组”或某个具体分组
  - 如果比赛已进入封榜时间，会展示封榜提示
  - 分题统计按 `category` 分组后再展示卡片
  - 分题统计区会明确说明当前只统计正式比赛解题，赛后练习不会计入公开分数和前三血
  - 每道题会同时展示前三血队伍名称，缺失时显示“暂无”
  - 公开比赛现在允许游客直接访问排行榜接口，不需要先登录也能查看公开总榜与分题统计

## 后续完善方向

这一版先实现最小可用的比赛榜单视图，并补上基础封榜能力，后续可以继续补：

# 管理端

当前管理端先实现了一个最小闭环页面：

- 路径：`/console/admin`
- 仅 `admin` / `super_admin` 角色可见
- 能力：
  - 创建比赛
  - 修改比赛名称、描述、公告、时间
  - 修改比赛状态、报名模式与公开性
  - 删除比赛
  - 导入比赛
  - 导出比赛
  - 审核参赛队伍状态（待审核 / 已通过 / 已拒绝）
  - 移除错误报名
  - 创建题目
  - 修改题目基础信息、提示、附件链接与计分参数
  - 从管理端删除题目
  - 将已有题目挂载到比赛
  - 查看比赛参赛队伍与当前得分
  - 查看某场比赛当前已挂载题目
  - 将题目从比赛中移除

## 当前设计原则

- 先复用现有后端 admin API
- 前端保持极简，只做表单输入和结果反馈
- 同步加载比赛、题目列表，方便管理时直接选择
- 选中比赛后直接查看当前挂题，减少管理切换成本
- 当前挂题列表使用独立的 admin 查询接口，不复用选手题目可见性逻辑
- 即使题目处于隐藏状态，管理员仍能在挂题视图里看到完整题面、提示和附件字段，便于核对配置
- 比赛基础信息与比赛状态拆成两个独立表单，避免误操作
- 比赛基础信息中单独加入“公告”字段，便于在比赛详情页面向选手展示临时规则或提醒
- 比赛状态编辑先只开放 `draft / active / ended`、`registration_mode` 与 `is_public`
- 报名模式当前支持：
  - `review`：提交报名后进入待审核
  - `auto_accept`：提交报名后直接获得参赛资格
- 比赛扩展配置当前支持：
  - `practice_mode`：标记比赛结束后是否继续保留练习入口
  - `writeup_required`：标记比赛是否要求赛后提交 Writeup
  - `writeup_deadline`：留空即可；如果填写，不能早于比赛 `end_time`
- 队伍人数上限当前支持：
  - `0`：不限制人数
  - 正整数：超过人数上限的队伍无法报名该比赛
- 封榜时间当前支持：
  - 留空：公开榜单不封榜
  - 指定时间：到达该时间后公开榜单冻结，但比赛仍可继续提交
- 参赛队伍卡片允许直接修改报名状态，便于管理员快速放行或拒绝报名申请
- 题目表单支持直接录入：
  - 题目描述
  - 提示列表（JSON 数组）
  - 附件链接（JSON 数组）
- 题目编辑先覆盖标题、描述、分类、难度、可见性与基础计分参数
- 题目删除放在已加载资源列表中，并在前端做二次确认
- 资源列表提供快捷按钮，可一键填充当前管理表单
- 参赛队伍列表先展示报名时间、状态、当前得分和解题数
- 删除比赛只清理当前比赛范围内的数据：
  - 报名记录
  - 比赛内解题记录
  - 比赛挂题关系
  - 题库中的原始题目不会被一起删除，方便后续复用
- 导出比赛会下载一个 ZIP 包：
  - 当前包内只有 `game.json`
  - 包含比赛基础信息与已挂载题目的完整配置
  - 当前导出版本为 `sauryctf.export.v2`
  - 也会包含 `practice_mode`、`writeup_required`、`writeup_deadline`
  - 外部附件链接会继续保留为 JSON 数组
  - 如果题目附件使用本地 `/attachments/**` 路径，这些文件也会一起打包进 ZIP
- 导入比赛当前支持上传管理端导出的 ZIP 包：
  - 接受 `sauryctf.export.v1` 和 `sauryctf.export.v2`
  - 必须包含 `game.json`
  - 导入后会新建一场 `draft` 比赛
  - 比赛的练习模式和 Writeup 配置会一起恢复
  - 题目会作为新的题库记录重新创建，再挂载到新比赛
  - `v2` 包里嵌入的本地附件会恢复到服务端 `./attachments`
  - 外部附件链接仍只按 URL 保存，不会在导入时主动下载
- 不在这一版里加入复杂的编辑、删除、批量操作

# 公开比赛页

当前公开比赛链路以 `/games` 和 `/games/{id}` 为主入口：

- 游客可以先浏览公开比赛
- 已有账号的选手可以登录后回到原比赛
- 新用户会先注册并准备队伍，再回到原比赛报名

## 页面入口

- `/games`
  - 用于浏览公开比赛列表
  - 顶部保持一条紧凑入口横幅，说明当前最适合的登录、注册、队伍或比赛入口
  - 单比赛卡片会合并展示当前账号的参赛状态与关键规则摘要
- `/games/{id}`
  - 用于查看单场比赛详情、报名状态、题目、排行榜和 Writeup
  - 页面会根据“未登录 / 没队伍 / 未报名 / 待审核 / 已通过”切换状态卡与操作入口
  - 顶部报名区优先合并为一张参赛上下文卡：主结论、状态摘要、下一步动作和主操作入口集中展示，不再拆成多块并列说明
  - 如果下一步仍在当前页上下文内完成，例如报名、切到题目、切到排行榜或切到 Writeup：
    - 顶部主按钮应直接执行当前页动作或切换标签，不再跳回同一路由造成“看起来没反应”的空跳转

## 跳转约定

- 从比赛详情页点击“登录”：
  - 会带上 `redirect=/games/{id}`
  - 登录成功后直接回到原比赛
- 从比赛详情页点击“注册”：
  - 也会带上 `redirect=/games/{id}`
  - 注册成功后不会直接回比赛，而是先进入 `/console/team?redirect=/games/{id}`
  - 创建队伍或加入队伍成功后，才会自动回到原比赛
- 从比赛详情页点击“去队伍页”：
  - 同样会保留当前比赛 `redirect`
  - 方便已登录用户处理完队伍后继续回到原比赛报名

## 公开浏览边界

- 游客可以看到：
  - 比赛基础信息
  - 题目标题、分类、分值、解题统计
  - 公开排行榜
- 游客不能直接看到：
  - 完整题面
  - 提示
  - 附件
  - 自己的报名与队伍状态

## 异常状态

- 如果比赛不存在，或当前账号暂时不能访问这场比赛：
  - 详情页会直接显示明确的不可用状态
  - 页面内会提供“返回比赛列表”和“重新加载”入口
  - 未登录用户也可以先登录后再重试

## 比赛限制展示

- 比赛详情页当前会集中展示这些和参赛决策最相关的限制：
  - 报名方式
  - 是否需要邀请码
  - 队伍人数上限
  - 分组规则
  - Writeup 要求
- 如果比赛里已经挂了动态题：
  - 概览区也会额外提示“每支队伍最多可同时运行多少个动态实例”
  - 这条信息用于提前展示队伍级动态实例数量限制

## 概览结构

- 比赛列表页：
  - 顶部只保留一条入口横幅，不再使用大块引导式提示框
  - 单比赛卡片把“我的参赛状态”和“规则摘要”合并为一块上下文摘要
  - 卡片底部继续保留主入口与辅助入口按钮

- 比赛信息：
  - 展示比赛正文和长期补充信息
- 规则摘要：
  - 用简短条目概括报名、封榜、练习模式和报名撤回规则
- 赛时通知：
  - 单独展示管理员赛中发布的公告，不与长期规则混排
- 参赛限制：
  - 当前改为和分组边界合并展示
  - 集中承接报名方式、邀请码、人数限制、Writeup、动态实例限制与分组规则
- 当前参赛上下文：
  - 集中展示账号、队伍、报名状态、提交权限、当前队伍和当前分组
- 实例接入信息：
  - 如果题目提供了入口、`host:port` 或连接命令，比赛详情页可以直接复制这些内容
- Writeup 区：
  - 只有在当前账号满足提交流程时才会展示编辑入口
  - 真正的编辑和提交统一在 `UModal` 中完成，正文只保留状态、备注和规则摘要
  - 未登录、未组队或报名未通过时，会直接显示对应入口
  - 更完整的提交流程与状态约束见 [writeup-workflow.md](/d:/Projects/sauryctf/docs/guide/writeup-workflow.md)
- 标签页顶部状态：
  - `题目 / 排行榜 / Writeup` 这三类标签页如果需要说明公开边界、封榜状态或提交权限，优先使用紧凑摘要头
  - 不再为同一标签页状态额外堆整块说明型 `UAlert`
- 高风险操作：
  - 退出当前报名
  - 销毁当前动态实例
  - 这两类操作现在统一先经过确认弹层，再真正调用后端接口
- Flag 提交区：
  - 只有在当前队伍已经具备可提交资格时才会展示输入框
  - 未登录、未组队或报名未通过时，会直接显示对应入口
- 空态展示：
  - 题目列表为空时，优先使用简短空态，不展开排查式说明
  - 公开榜单为空时，按“全局无记录 / 当前分组无记录”区分即可
  - 分题统计为空时，直接说明暂无统计数据
  - Writeup 审核备注为空时，保留独立状态区块即可

## 推荐验证顺序

1. 先打开 `/games`
2. 进入某场公开比赛详情页
3. 用游客视角确认只显示公开信息
4. 点击“登录”或“注册”，确认会保留原比赛返回路径
5. 处理好队伍后回到原比赛完成报名
6. 报名通过并开赛后，再确认题面、提交入口和实例入口逐步开放

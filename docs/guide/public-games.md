# 公开比赛页

当前公开比赛链路以 `/games` 和 `/games/{id}` 为主入口，按选手实际参赛路径组织：

- 游客可以先浏览公开比赛
- 已有账号的选手可以直接登录后回到原比赛
- 新用户会先注册并准备队伍，再回到原比赛继续报名

## 当前入口

- `/games`
  - 用于浏览公开比赛列表
  - 会展示当前账号的参赛状态摘要和下一步推荐
- `/games/{id}`
  - 用于查看单场比赛详情、报名状态、题目、排行榜和 Writeup
  - 页面会根据“未登录 / 没队伍 / 未报名 / 待审核 / 已通过”切换引导文案

## 当前跳转约定

- 从比赛详情页点击“登录”：
  - 会带上 `redirect=/games/{id}`
  - 登录成功后直接回到原比赛
- 从比赛详情页点击“注册”：
  - 也会带上 `redirect=/games/{id}`
  - 注册成功后不会直接回比赛，而是先进入 `/console/team?onboarding=created&redirect=/games/{id}`
  - 创建队伍或加入队伍成功后，才会自动回到原比赛
- 从比赛详情页点击“去队伍页”：
  - 同样会保留当前比赛 `redirect`
  - 方便已登录用户处理完队伍后继续回到原比赛报名

## 当前公开浏览边界

- 游客可以看到：
  - 比赛基础信息
  - 题目标题、分类、分值、解题统计
  - 公开排行榜
- 游客当前不能直接看到：
  - 完整题面
  - 提示
  - 附件
  - 自己的报名与队伍状态

## 当前异常状态

- 如果比赛不存在，或当前账号暂时不能访问这场比赛：
  - 详情页会直接显示明确的不可用状态
  - 页面内会提供“返回比赛列表”和“重新加载”入口
  - 未登录用户也可以顺手先去登录后再重试

## 当前比赛限制展示

- 比赛详情页当前会集中展示这些和参赛决策最相关的限制：
  - 报名方式
  - 是否需要邀请码
  - 队伍人数上限
  - 分组规则
  - Writeup 要求
- 如果比赛里已经挂了动态题：
  - 概览区也会额外提示“每支队伍最多可同时运行多少个动态实例”
  - 这条语义用于提前说明队伍级动态实例数量限制

## 当前概览结构

- 比赛说明：
  - 展示比赛正文和长期规则补充
- 规则摘要：
  - 用简短条目概括报名、封榜、练习模式和报名撤回规则
- 赛时通知：
  - 单独展示管理员赛中发布的公告，不与长期规则混排
- 参赛限制：
  - 集中展示报名方式、邀请码、人数限制、Writeup 和动态实例限制
- 队伍与分组：
  - 单独展示当前队伍、当前分组和分组分配规则
- 实例接入信息：
  - 如果题目提供了入口、`host:port` 或连接命令，比赛详情页可以直接复制这些内容

## 推荐验证顺序

1. 先打开 `/games`
2. 进入某场公开比赛详情页
3. 用游客视角确认只显示公开信息
4. 点击“登录”或“注册”，确认会保留原比赛返回路径
5. 处理好队伍后回到原比赛完成报名
6. 报名通过并开赛后，再确认题面、提交入口和实例入口逐步开放

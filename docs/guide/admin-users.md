# 用户管理

`/console/admin/users` 用于处理基础账号权限与状态调整。

## 当前能力

- 查看用户列表
- 按关键词、角色、状态筛选
- 调整用户角色
- 调整用户状态
- 保存前通过确认弹层复核变更
- 支持从审计日志携带 `highlight_user_id` 回到目标账号记录

## 页面约定

- 页首只保留一块紧凑摘要，说明当前页处理范围与权限边界
- 不再额外堆整块说明型 `UAlert`
- 低频高影响动作继续通过确认弹层提交
- 列表主体优先保留筛选、当前状态、草稿改动和保存入口
- 如果从审计日志跳回用户页，应以紧凑上下文提示和列表高亮承接目标账号，而不是新增说明面板或额外引导页

## 当前限制

- 当前登录账号不能修改自己的角色
- 当前登录账号不能把自己改为封禁
- 非更高权限账号不能越权调整受限目标

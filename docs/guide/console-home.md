# 控制台首页

当前 `/console` 已从占位页调整为可用的导航首页。

## 当前能力

- 展示我的队伍状态
- 如果是新注册用户直接进入控制台，顶部会出现“先准备队伍”的首次引导
- 展示一个面向选手的“下一步”卡片，直接提示当前最推荐的操作顺序
- 展示公开比赛统计
- 展示“我的比赛”入口，按报名状态聚合当前用户最需要处理的比赛
- 展示“我的比赛公告”，汇总当前账号已关联比赛的最新公告
- 展示最近比赛列表
- 提供最近比赛快捷入口
- 无数据区域统一使用正式空态组件，并直接给出下一步操作入口
- 管理员可直接跳转到 `/console/admin`
- 如果管理员首次进入控制台且当前还没有任何比赛：
  - 顶部会出现一张“空库首次启动”引导卡
  - 会直接给出 3 步最短路径：
    - 先创建公开比赛
    - 再创建题目并挂题
    - 最后回公开页验证选手链路
- 管理员现在也会在首页看到一组轻量待办摘要：
  - 最近 5 场比赛里的待审核报名
  - 最近 5 场比赛里的待审 Writeup
  - 最近 5 场比赛里的最新公告
  - 最近 5 场比赛里的最新提交记录
  - 最近 5 场比赛里的重复错误 Flag 线索
- 管理员现在也可以直接在控制台首页快速切换最近比赛状态：
  - `draft -> active`
  - `active -> ended`
- 普通用户如果误入 `/console/admin`，会被前端中间件直接带回 `/console`

## 当前设计

- 只复用现有接口：
  - `GET /api/teams/my`
  - `GET /api/games`
  - `GET /api/games/{id}/participation`
- 前端保持 Nuxt UI 极简卡片式布局
- 首页里的比赛、队伍、管理待办等无数据区域统一使用正式空态：
  - 会明确说明当前为什么为空
  - 会直接给出下一步入口，例如建赛、队伍管理、公告区或审计页
- 注册后会优先直达 `/console/team?onboarding=created`，避免新用户先落到控制台首页再多点一次
- 管理员待办当前已经收敛为一个轻量聚合接口：
  - `GET /api/admin/dashboard/summary`
  - 返回最近 5 场比赛、待审核报名、待审 Writeup、最新公告、最近提交与可疑线索摘要
  - 控制台首页不再逐场拉取报名 / Writeup / 公告接口
  - 最近提交与可疑线索复用现有提交查询能力做轻量聚合，不额外引入新的监控接口
- “下一步”卡片当前会固定覆盖最小参赛路径：
  - 先准备队伍
  - 再选择比赛
  - 最后处理当前最紧急的待办，例如待审核、补交 Writeup 或进入比赛
- “我的比赛”当前会优先突出：
  - 待补 Writeup
  - 待审核 / 已拒绝
  - 已报名且正在进行中的比赛
  - 尚未报名但仍可进入的公开比赛
- “我的比赛公告”当前会：
  - 只汇总当前账号已关联比赛的公告
  - 按发布时间倒序展示最近 6 条
  - 每条公告都可直接跳回对应比赛详情页
- 管理员快捷操作区当前会优先覆盖本地最常见的建赛闭环：
  - 打开管理端继续配置
  - 直接打开公开比赛详情页验证展示
  - 将 `draft` 比赛一键切为 `active`
  - 将 `active` 比赛一键切为 `ended`
- 首页现在也会额外承担一个“首次建赛前台引导”职责：
  - 不依赖新的后端接口
  - 只根据“管理员身份 + 当前没有任何比赛”判断为高概率空库场景
  - 目标是让 `admin / sauryctf` 首次登录后，不用再猜下一步先去哪里

## 后续可继续补充

- 可疑线索的进一步筛选与备注
- 更细的比赛事件时间线

## 当前公开可见性

- 公开比赛列表只展示：
  - `is_public = true`
  - 且不处于 `draft` 的比赛
- 草稿比赛仍然保留在管理端中，方便管理员继续配置

# 控制台首页

当前 `/console` 是统一的控制台首页。

## 当前能力

- 展示我的队伍状态
- 展示一个面向选手的“当前状态”卡片，直接汇总当前最需要处理的事项
- 展示公开比赛统计
- 展示“我的比赛”入口，按报名状态聚合当前用户最需要处理的比赛
- 展示“我的比赛公告”，汇总当前账号已关联比赛的最新公告
- 展示最近比赛列表
- 提供最近比赛快捷入口
- 无数据区域统一使用正式空态组件，并直接给出对应操作入口
- 管理员可直接跳转到 `/console/admin`
- 管理员现在也会在首页看到一组轻量待办摘要：
  - 最近 5 场比赛里的待审核报名
  - 最近 5 场比赛里的待审 Writeup
  - 最近 5 场比赛里的最新公告
  - 最近 5 场比赛里的最新提交记录
  - 最近 5 场比赛里的重复错误 Flag 线索
  - 首页现在只详细展开“待审核报名 / 待审 Writeup”
  - 其余公告、提交与可疑线索改为更紧凑的“赛时摘要”，优先承担总览和跳转
- 管理员现在也可以直接在控制台首页快速切换最近比赛状态：
  - `draft -> active`
  - `active -> ended`
- 普通用户如果误入 `/console/admin`，会被前端中间件直接带回 `/console`

## 当前设计

- 只复用现有接口：
  - `GET /api/teams/my`
  - `GET /api/games`
  - `GET /api/games/{id}/participation`
- 前端保持 Nuxt UI 极简卡片式布局
- 首页里的比赛、队伍、管理待办等无数据区域统一使用正式空态：
  - 会明确说明当前为什么为空
  - 会直接给出对应入口，例如建赛、队伍管理、公告区或审计页
- 注册后会优先直达 `/console/team`，避免新用户先落到控制台首页再多点一次
- 管理员待办当前已经收敛为一个轻量聚合接口：
  - `GET /api/admin/dashboard/summary`
  - 返回最近 5 场比赛、待审核报名、待审 Writeup、最新公告、最近提交与可疑线索摘要
  - 控制台首页不再逐场拉取报名 / Writeup / 公告接口
  - 最近提交与可疑线索复用现有提交查询能力做轻量聚合，不额外引入新的监控接口
- “当前状态”卡片当前覆盖最小参赛路径：
  - 先准备队伍
  - 再选择比赛
  - 最后处理当前最紧急的待办，例如待审核、补交 Writeup 或进入比赛
- 首页现在进一步压缩了重复入口：
  - 管理员相关深入口不再在右侧重复堆叠
  - “常用操作”只保留队伍、比赛和下一场比赛入口
  - 右侧最后一张卡片更聚焦账号安全与基础导航
- 队伍页的未组队视图也保持同样的克制表达：
  - 创建/加入动作继续放在弹层里
  - 页面主体只保留入口状态、队伍规则和返回比赛相关信息
  - 避免再加一整块列表式“说明”文本，优先使用结构化状态摘要
- 默认管理员如果仍在使用初始密码：
  - `/console` 顶部会显示账号安全提醒
  - `/console/account` 会继续显示更强的改密提示
- “我的比赛”当前会优先突出：
  - 待补 Writeup
  - 待审核 / 已拒绝
  - 已报名且正在进行中的比赛
  - 尚未报名但仍可进入的公开比赛
- 控制台首页与公开比赛页当前复用同一套参与状态解释：
  - `未加入队伍 / 可报名 / 待审核 / 已拒绝 / 已报名 / 待补 Writeup / Writeup 待审`
  - 比赛阶段统一按 `draft / before_start / active / ended` 判断，避免首页与公开页对“未开始”状态理解不一致
- 控制台首页里的比赛统计、最近比赛和管理员快捷操作现在也统一按比赛阶段而不是原始 `status` 展示：
  - 已发布但未到开始时间的比赛会显示为 `未开始`
  - 只有真正处于比赛窗口内的比赛才会计入 `进行中比赛`
  - 页面停留期间会定时刷新阶段判断，避免长时间打开后状态过期
- 管理员视角下首页会同时拿到全量比赛数据，但选手向区块仍只基于公开可见比赛：
  - `公开比赛统计`
  - `最近比赛`
  - `下一场比赛`
  - 这些区块不会把私有或草稿比赛混进去
- “我的比赛公告”当前会：
  - 只汇总当前账号已关联比赛的公告
  - 按发布时间倒序展示最近 6 条
  - 每条公告都可直接跳回对应比赛详情页
- 管理员快捷操作区当前优先覆盖常见建赛闭环：
  - 打开管理端继续配置
  - 直接打开公开比赛详情页验证展示
  - 将 `draft` 比赛一键切为 `active`
  - 将 `active` 比赛一键切为 `ended`

- 草稿比赛仍然保留在管理端中，方便管理员继续配置

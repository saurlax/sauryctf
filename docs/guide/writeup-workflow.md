# Writeup 流程

当前平台已经提供一条最小可用的 `Writeup` 闭环，覆盖选手提交、管理员审核与赛后导出。

## 选手侧

- 页面位置：
  - `/games/{id}` 的 `Writeup` 标签
- 当前接口：
  - `GET /api/games/{id}/writeup`
  - `PUT /api/games/{id}/writeup`
- 当前提交前提：
  - 当前比赛开启 `writeup_required`
  - 当前队伍报名状态为 `accepted`
  - 当前比赛 `writeup_deadline` 未过

## 当前状态

- 比赛详情页会明确区分这些情况：
  - 当前比赛未启用 `Writeup`
  - 尚未登录
  - 尚未组队
  - 尚未报名
  - 报名待审核
  - 已过 `Writeup` 截止时间
  - 当前可提交
  - 已提交待审核
  - 已驳回，可重新提交
  - 已通过

- 当前标签页保持双栏结构：
  - 左侧保留正文编辑区
  - 右侧集中展示提交状态、截止时间、提交时间、审核时间与审核备注
  - 不再额外叠加单独的规则列表或重复状态卡

## 提交规则

- `Writeup` 按队伍保存：
  - 同一支队伍在同一场比赛里只有一份正文
  - 重新提交会覆盖已有内容
- 前端当前会先校验：
  - 正文非空
- 最终是否允许提交，仍以后端返回为准：
  - 比赛是否要求 `Writeup`
  - 当前队伍是否已通过报名
  - 截止时间是否已过

## 管理侧

- 页面位置：
  - `/console/admin` 的 `Writeup 审核` 区
- 当前能力：
  - 查看某场比赛的全部 `Writeup`
  - 修改审核结果为：
    - `待审核`
    - `已通过`
    - `已驳回`
  - 填写审核备注
  - 导出当前比赛的 `Writeup`

## 审核交互

- 审核入口保持在页面主体：
  - 列表中先显示当前状态、提交时间和审核草稿摘要
  - 真正编辑在 `UModal` 中完成
- 保存前会进入确认弹层：
  - 复核目标队伍
  - 复核审核结果
  - 复核审核备注

## 导出结果

- 管理端可以为单场比赛导出单独的 `Writeup` 包
- 当前导出内容包括：
  - `writeups.json`
  - `writeups.csv`
  - `writeups/team-*.md`

## 当前约束

- 如果比赛未开启 `writeup_required`：
  - 选手侧只展示状态说明
  - 不开放正文编辑入口
- 如果 `writeup_deadline` 已过：
  - `can_submit = false`
  - 选手侧不再允许更新正文
- 只有 `accepted` 队伍可以进入正式提交流程
- 管理员审核后的备注会直接显示在选手侧状态区

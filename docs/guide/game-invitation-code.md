# 比赛邀请码

当前比赛支持一层可选的邀请码门槛，行为参考了 GZCTF 常见的“公开可见、报名受控”办赛方式。

## 当前能力

- 管理员现在可以在比赛创建、比赛编辑、比赛设置里填写 `invitation_code`
- 留空表示不启用邀请码
- 只要填写了邀请码：
  - 公开比赛列表和比赛详情仍然可浏览
  - 公开接口只会返回 `invitation_required = true`
  - 不会返回邀请码明文
  - 队伍报名时必须提交正确的邀请码
- 比赛详情页会在这些条件同时满足时显示邀请码输入框：
  - 当前用户已登录
  - 当前用户已经有队伍
  - 当前队伍还没有报名这场比赛
  - 当前比赛启用了邀请码

## 管理端约束

- `GET /api/games?all=true`
- `GET /api/games/{id}?all=true`

这两个扩展读取入口现在只允许管理员访问。

这样做的目的是让管理端继续复用现有 `/api/games` 读取比赛全量信息，同时避免把草稿赛、私有赛和比赛邀请码明文暴露给普通用户。

## 本地联调建议

1. 用管理员创建一场 `is_public = true` 的比赛
2. 填一个 `invitation_code`
3. 把比赛状态切到 `active`
4. 用普通账号登录并准备队伍
5. 打开比赛详情页，确认能看到邀请码输入框
6. 先用错误邀请码报名，确认返回失败
7. 再用正确邀请码报名，确认报名成功或进入待审核

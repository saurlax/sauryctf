# 审计日志

平台现在提供一份最小可用的管理端审计日志视图：

- 路径：`/console/admin/audit`
- 对象范围：
  - 用户
  - 比赛
  - 题目
- 当前已记录的管理动作：
  - 更新用户角色或状态
  - 创建比赛
  - 更新比赛
  - 删除比赛
  - 导入比赛包
  - 导出比赛包
  - 导出榜单
  - 导出 Writeup
  - 导出提交记录
  - 向比赛挂载题目
  - 从比赛移除题目
  - 更新报名状态
  - 移除报名
  - 审核 Writeup
  - 创建比赛公告
  - 编辑比赛公告
  - 删除比赛公告
  - 手动销毁实例租约
  - 创建题目
  - 更新题目
  - 删除题目

## 页面用途

- 快速确认最近是谁改过比赛或题目
- 回看某次权限调整是否已经生效
- 在多人协作时保留最基础的管理操作痕迹

## 当前筛选

- 按关键词搜索摘要、详情、操作者或对象 ID
- 按操作者 ID 筛选
- 按对象类型筛选
- 按动作类型筛选
- 按记录条数筛选
- `操作者 ID / 对象类型 / 动作类型 / 记录条数` 当前都会直接进入后端查询条件
- `关键词` 继续保持页内快速过滤，便于在已拉取结果上二次收窄
- 当前筛选状态会同步到页面 query：
  - 刷新页面后仍可恢复当前筛选
  - 可以直接分享带筛选条件的审计日志链接
  - 浏览器前进、后退会承接对应筛选状态

## 当前展示约定

- 页面优先展示正式中文动作名，不直接把原始 action key 暴露为主要信息
- `detail` 默认按需展开，避免在长列表里直接堆满原始细节字段
- 日志条目现在也会提供“回到对象”入口：
  - 用户日志可直接跳到用户管理页
  - 比赛日志可直接跳回对应比赛的设置、报名、公告、Writeup、实例监控或挂题区
  - 如果日志详情里带了 `announcement_id`、`team_id` 或 `challenge_id`，页面会继续承接到更具体的公告编辑、目标队伍审核或题目上下文
  - `admin.game.update_participation`、`admin.game.remove_participation`、`admin.game.review_writeup` 这类记录现在会优先直开对应队伍的审核 `UModal`
  - 题目日志可直接跳回题目维护区，并在可能时带上关联比赛上下文
  - `admin.game.destroy_instance_lease` 这类实例运维日志当前会回到 `/console/admin?game_id=...&section=%23monitoring`
  - 如果详情里还带了 `challenge_id`，管理端会继续保留题目级高亮，直接落到实例运维明细而不是总览标签
- 当前已覆盖的动作名包括：
  - 导入比赛包 / 导出比赛包 / 导出榜单 / 导出 Writeup / 导出提交记录
  - 发布公告 / 编辑公告 / 删除公告
  - 挂载题目 / 从比赛移除题目
  - 更新报名状态 / 移除报名
  - 审核 Writeup
  - 手动销毁实例租约
- 动作类型筛选当前直接覆盖这些已接入动作，适合快速只看比赛变更、题目变更或账号调整。

这一版先保持极简，只覆盖最常见的管理动作与基础查询需求。

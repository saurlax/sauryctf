# 审计日志

平台现在提供一份最小可用的管理端审计日志视图：

- 路径：`/console/admin/audit`
- 对象范围：
  - 用户
  - 比赛
  - 题目
- 当前已记录的管理动作：
  - 更新用户角色或状态
  - 创建比赛
  - 更新比赛
  - 删除比赛
  - 导入比赛包
  - 导出比赛包
  - 导出榜单
  - 导出 Writeup
  - 导出提交记录
  - 向比赛挂载题目
  - 从比赛移除题目
  - 更新报名状态
  - 移除报名
  - 审核 Writeup
  - 创建比赛公告
  - 删除比赛公告
  - 手动销毁实例租约
  - 创建题目
  - 更新题目
  - 删除题目

## 页面用途

- 快速确认最近是谁改过比赛或题目
- 回看某次权限调整是否已经生效
- 在多人协作时保留最基础的管理操作痕迹

## 当前筛选

- 按关键词搜索摘要、详情、操作者或对象 ID
- 按操作者 ID 筛选
- 按对象类型筛选
- 按动作类型筛选
- 按记录条数筛选

## 当前展示约定

- 页面优先展示正式中文动作名，不直接把原始 action key 暴露为主要信息
- `detail` 默认按需展开，避免在长列表里直接堆满原始细节字段
- 当前已覆盖的动作名包括：
  - 导入比赛包 / 导出比赛包 / 导出榜单 / 导出 Writeup / 导出提交记录
  - 发布公告 / 删除公告
  - 挂载题目 / 从比赛移除题目
  - 更新报名状态 / 移除报名
  - 审核 Writeup
  - 手动销毁实例租约
- 动作类型筛选当前直接覆盖这些已接入动作，适合快速只看比赛变更、题目变更或账号调整。

这一版先保持极简，只覆盖最常见的管理动作与基础查询需求。

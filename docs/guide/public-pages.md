# 公开页

公开页当前以“状态可读、操作克制”为主，避免做成流程引导页。

## 文案约定

- 首页入口使用平台化命名，例如：
  - `管理入口`
  - `选手账号`
  - `公开比赛`
- 比赛详情页优先展示状态摘要，而不是编号步骤：
  - `账号状态`
  - `队伍状态`
  - `报名状态`
  - `提交权限`
- 状态提示标题避免使用 `当前状态：...` 这类前缀，直接使用结论式标题：
  - `需要先登录`
  - `报名待审核`
  - `比赛已结束`

## 交互约定

- 未登录访客仍可浏览公开比赛基础信息、题目标题与排行榜。
- 登录、注册、队伍管理等动作可以保留，但只作为明确入口，不展开成长流程说明。
- 比赛详情页右侧面板优先承担：
  - 当前行动
  - 参赛状态
  - 参赛限制
  - 队伍与分组
- 不再保留“第 1 步 / 第 2 步”式参与说明卡片。
- 空态按钮与受限状态按钮优先使用明确入口动作：
  - `登录账号`
  - `管理队伍`
  - `查看比赛概览`

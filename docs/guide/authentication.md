# 登录与注册

当前前端已经提供独立的 `/login` 与 `/register` 页面，右上角导航也固定提供两个入口：

- `登录`
- `注册`

## 默认管理员

只有在 `users` 表完全为空时，后端启动才会自动创建默认管理员：

- 用户名：`admin`
- 密码：`sauryctf`

一旦数据库里已经有任意用户：

- 后端不会再补建 `admin`
- 首页和 `/login` 也不会继续把默认口令当成常规入口提示
- `/api/auth/setup-status` 会立即切到 `bootstrap_admin_available = false`

最容易误判的点只有一个：

- `admin / sauryctf` 不是“系统保底账号”，而是“仅空库首次启动可用的引导账号”

## 普通用户链路

- 直接访问 `/register` 注册成功后，会自动登录并直接进入 `/console/team?onboarding=created`
- 队伍页会立刻提示先创建或加入队伍，避免新用户刚注册完还不知道下一步该做什么
- `/register` 现在也会额外显示一张“注册后下一步”卡片：
  - 普通场景会明确提示“先进队伍页、准备队伍、再去比赛”
  - 从比赛页跳来时，会明确提示“先准备队伍，再回原比赛继续报名或提 Flag”
- `/register` 页面右侧现在也会固定展示一组 3 步 onboarding 卡片：
  - 先创建账号
  - 再准备队伍
  - 最后回到比赛继续报名
- 如果是从比赛详情页跳到 `/register?redirect=/games/{id}`：
  - 注册成功后会先进入 `/console/team?onboarding=created&redirect=/games/{id}`
  - 队伍页会同时保留“账号已创建”的提示和“完成后回到原比赛”的提示
  - 创建队伍或加入队伍成功后，会自动回到原比赛

## 登录链路

- `/login` 支持用户名或邮箱登录
- 如果当前链接带有 `redirect` 查询参数，登录成功后会优先跳回该地址
- 如果没有 `redirect`：
  - 空库管理员引导场景默认进入 `/console/admin`
  - 其他账号默认进入 `/console`
- `/login` 现在会额外显示一张“登录后下一步”卡片：
  - 从比赛页跳来时，会明确提示“先回到原比赛，再补队伍或报名”
  - 空库管理员场景，会明确提示“先进管理端、建赛、再测普通选手链路”
  - 普通账号场景，会明确提示“先进控制台，再准备队伍，再回比赛页”
- `/login` 页面右侧现在也会显示几个固定快捷入口：
  - 空库场景下的默认管理员提示
  - 公开比赛列表
  - 注册入口
- Nuxt 开发环境下，认证相关首屏请求现在区分了两条访问路径：
  - 浏览器侧继续走同源 `/api/**`，保留 Cookie 与 dev proxy 行为
  - Nuxt 服务端渲染时改为直接访问 `http://127.0.0.1:8080/api`
  - 这样像 `/login`、首页这类会在首屏拉取 `/api/auth/setup-status` 的页面，就不会再把 `/api/**` 误判成 Nuxt 自己的内部路由

## 推荐验证顺序

1. 空库启动后先用 `admin / sauryctf` 登录
2. 创建一场公开比赛并激活
3. 再注册普通账号
4. 创建或加入队伍
5. 回到比赛详情继续报名、提 Flag、看榜

# 题目维护

## 基础字段

- 题目维护继续集中在 `/console/admin` 的弹层里完成
- 题目基础信息至少包括：
  - 名称
  - 题面
  - `Flag`
  - `Flag 格式`
  - 分类与分值

## Flag 格式

- `Flag 格式` 用于公开比赛页展示提交示例
- 如果题目已经配置 `flag_format`：
  - 管理端列表应直接显示该格式
  - 比赛页提交输入框应复用该格式作为占位提示
- 不再在选手侧页面写死统一的 `flag{...}` 占位

## 维护约定

- 题目录入、编辑、模板填充保持极简，不额外叠加说明卡片
- 如果题目有实例接入信息，可继续通过实例预览核对入口、端口和运行配置
- 如果题目使用动态运行时：
  - 可在 `container_spec.runtime.env` 里声明最小环境变量映射
  - 当前主要用于本地 Docker provider 把题目运行参数传入容器
  - 管理端实例预览会直接展示这些键值对，便于在保存前核对

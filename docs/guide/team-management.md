# 队伍管理

`/console/team` 提供比赛报名前后所需的基础队伍管理能力。

## 当前能力

- 创建队伍
- 通过邀请码加入队伍
- 通过 `invite` 查询参数自动预填邀请码
- 复制邀请码或邀请链接分享给队友
- 从比赛详情页跳转到 `/console/team?redirect=...` 时，创建或加入成功后自动返回原比赛
- 普通成员主动退出队伍
- 队长移除其他成员
- 队长将队长身份移交给其他成员
- 队长重置邀请码，旧邀请码立即失效
- 显示当前锁定中的比赛，并可直接跳回比赛详情页

## 页面状态

- 未加入队伍时，页面会提示当前需要先完成队伍准备
- 未加入队伍时，创建队伍与加入队伍统一通过弹层完成，页面主体只保留入口、规则说明与比赛返回提示
- 如果当前来自比赛页，页面会保留返回地址，便于完成组队后继续原比赛流程
- 如果当前链接同时带有 `invite` 与 `redirect`：
  - 页面应只保留一组合并后的提示
  - 明确说明“已识别邀请码”以及“加入后会返回原比赛”
- 已加入队伍后，页面会根据当前状态提示是否适合返回比赛、继续邀请成员或保持队伍结构稳定
- 高影响操作会通过确认弹层执行，包括退队、移除成员、移交队长和重置邀请码

## 当前约束

- 创建者自动成为队长
- 队长不能直接退出队伍
- 队长需要退出时，应先移交队长身份
- 非成员不能借由已知 `team_id` 对其他队伍执行退队或移除成员操作
- 队伍一旦通过任意一场未结束比赛的报名，会进入临时锁定状态
- 锁定期间不能新增成员、退队、移除成员或移交队长
- 锁定期间队长仍可重置邀请码，用于停用旧入口

## 建议用法

- 队长优先分享邀请链接，减少队友手动输入邀请码
- 队伍组建完成后，再统一前往比赛详情页完成报名
- 如果邀请码已扩散给错误对象，可先重置邀请码再继续邀请
- 如果当前来自比赛详情页，直接在队伍页完成创建或加入即可，无需手动返回比赛入口

# 队伍管理

`/console/team` 提供比赛报名前后所需的基础队伍管理能力。

## 当前能力

- 创建队伍
- 通过邀请码加入队伍
- 通过 `invite` 查询参数自动预填邀请码
- 复制邀请码或邀请链接分享给队友
- 从比赛详情页跳转到 `/console/team?redirect=...` 时，创建或加入成功后自动返回原比赛
- 普通成员主动退出队伍
- 队长移除其他成员
- 队长将队长身份移交给其他成员
- 队长重置邀请码，旧邀请码立即失效
- 显示当前锁定中的比赛，并可直接跳回比赛详情页

## 页面状态

- 未加入队伍时，页面会提示当前需要先完成队伍准备
- 未加入队伍时，创建队伍与加入队伍统一通过弹层完成，页面主体优先收敛为一个主入口卡片：
  - 保留入口按钮
  - 保留比赛返回状态或邀请上下文提示
  - 将基础队伍规则直接并入同一张入口卡
  - 不再并排放多张重复解释同一流程的提示卡或摘要卡
  - 如果需要说明当前邀请或返回上下文，也优先用一块紧凑状态摘要承接
- 如果当前来自比赛页，页面会保留返回地址，便于完成组队后继续原比赛流程
- 如果当前带着 `invite` 邀请链接进入且还没有队伍：
  - 页面会直接打开“加入队伍”弹层
  - 邀请码会自动带入，无需再次点击入口
- 如果当前只带着比赛 `redirect` 进入且还没有队伍：
  - 页面会直接打开“创建队伍”弹层
  - 完成创建后会自动回到原比赛
- 如果当前链接同时带有 `invite` 与 `redirect`：
  - 页面应只保留一组合并后的提示
  - 明确说明“已识别邀请码”以及“加入后会返回原比赛”
- 已加入队伍后，页面会根据当前状态提示是否适合返回比赛、继续邀请成员或保持队伍结构稳定
- 已加入队伍后的顶部状态提示应尽量保持为紧凑摘要卡：
  - 当前下一步作为主结论
  - 邀请上下文或比赛返回地址作为同卡内的补充摘要
  - 不再连续堆叠两块同类提示卡分别重复说明按钮和返回路径
  - 如果仍需保留邀请或回跳上下文，优先改成同卡内的摘要行，而不是再嵌一块次级说明框
- 已加入队伍后，右侧信息区优先展示可操作边界与锁定约束：
  - 保留角色、邀请码状态、可移除成员、可移交队长、锁定比赛数量等摘要
  - 将“维护建议 / 参赛建议”收敛为一段边界说明，不再叠加额外提示卡
- 未加入队伍时，队伍规则、创建后角色和回跳去向也应优先收敛为同卡内的摘要行，不再额外追加第二段说明块
- 高影响操作会通过确认弹层执行，包括退队、移除成员、移交队长和重置邀请码

## 当前约束

- 创建者自动成为队长
- 队长不能直接退出队伍
- 队长需要退出时，应先移交队长身份
- 非成员不能借由已知 `team_id` 对其他队伍执行退队或移除成员操作
- 队伍一旦通过任意一场未结束比赛的报名，会进入临时锁定状态
- 锁定期间不能新增成员、退队、移除成员或移交队长
- 锁定期间队长仍可重置邀请码，用于停用旧入口

## 建议用法

- 队长优先分享邀请链接，减少队友手动输入邀请码
- 队伍组建完成后，再统一前往比赛详情页完成报名
- 如果邀请码已扩散给错误对象，可先重置邀请码再继续邀请
- 如果当前来自比赛详情页，直接在队伍页完成创建或加入即可，无需手动返回比赛入口

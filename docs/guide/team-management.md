# 队伍管理

当前 `/console/team` 已支持最小可用的队伍管理流程。

## 当前能力

- 创建队伍
- 创建成功后，接口会立即返回完整的队长与首位成员信息，前端无需额外刷新即可渲染队伍卡片
- 通过邀请码加入队伍
- 通过 `invite` 查询参数自动预填邀请码
- 复制邀请码分享给队友
- 复制邀请链接，队友打开 `/console/team?invite=...` 后可直接加入
- 普通队员主动退出队伍
- 队长移除其他成员
- 显示当前锁定中的比赛，解释为什么当前不能调整队伍
- 显示当前成员数、队伍锁定状态、锁定比赛数，并支持从锁定比赛卡片直接跳回比赛详情页

## 当前约束

- 创建者自动成为队长
- 队长当前不能直接退出队伍
- 如果队伍已经通过任意一场仍未结束的比赛报名，队伍会被临时锁定：
  - 不能再通过邀请码加入新成员
  - 普通成员不能退队
  - 队长不能移除成员
- 移除成员使用现有接口 `DELETE /api/teams/{teamId}/members/{memberId}`

## 当前前端原则

- 保持 Nuxt UI 极简布局
- 队长和队员看到的操作保持明确分层
- 优先服务“创建队伍 -> 邀请队友 -> 报名比赛”的主流程
- 如果队伍已经被比赛锁定，优先直接展示锁定原因和关联比赛，而不是让用户靠接口报错理解规则

## 推荐使用方式

- 队长创建队伍后，优先直接复制邀请链接发给队友，而不是单独转发邀请码
- 队友打开邀请链接后，会自动填入邀请码，只需要登录并确认加入
- 队伍组建完成后，再统一前往比赛详情页完成报名，避免报名通过后再改成员导致锁队

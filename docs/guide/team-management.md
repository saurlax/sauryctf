# 队伍管理

当前 `/console/team` 已支持最小可用的队伍管理流程。

## 当前能力

- 创建队伍
- 创建成功后，接口会立即返回完整的队长与首位成员信息，前端无需额外刷新即可渲染队伍卡片
- 通过邀请码加入队伍
- 通过 `invite` 查询参数自动预填邀请码
- 复制邀请码分享给队友
- 复制邀请链接，队友打开 `/console/team?invite=...` 后可直接加入
- 如果是从比赛详情页跳转到 `/console/team?redirect=...`：
  - 页面会提示当前来自比赛报名流程
  - 创建队伍或加入队伍成功后，会自动回到原比赛页
  - 复制邀请链接时也会一并保留这个返回地址，方便整队一起回到同一场比赛
- 如果是刚注册完成后进入 `/console/team?onboarding=created&redirect=...`：
  - 页面会同时提示“账号已经创建完成”和“完成队伍准备后会自动回到原比赛”
  - 这样首登用户不会在建队前丢掉原来的比赛上下文
- 普通队员主动退出队伍
- 队长移除其他成员
- 队长可将队伍管理权移交给其他成员
- 队长可重置当前队伍的邀请码，旧邀请码会立即失效
- 显示当前锁定中的比赛，解释为什么当前不能调整队伍
- 显示当前成员数、队伍锁定状态、锁定比赛数，并支持从锁定比赛卡片直接跳回比赛详情页
- 如果当前已经有队伍：
  - 页面会额外显示一条“当前下一步”提示
  - 会根据场景切换为：
    - 从比赛页跳来时，优先提示“回原比赛继续报名”
    - 队长刚建好单人队伍时，优先提示“邀请队友或直接去报名”
    - 队伍已锁定时，优先提示“先回比赛继续参赛，不再调整成员”
    - 普通成员已在队伍里时，优先提示“直接去比赛页继续参赛”

## 当前约束

- 创建者自动成为队长
- 队长当前不能直接退出队伍
- 如果队长需要退出队伍，应先把队长身份移交给其他成员
- 如果队伍已经通过任意一场仍未结束的比赛报名，队伍会被临时锁定：
  - 不能再通过邀请码加入新成员
  - 普通成员不能退队
  - 队长不能移除成员
  - 队长不能移交管理权
  - 队长仍可重置邀请码，用于停用已经扩散的旧邀请码
- 移除成员使用现有接口 `DELETE /api/teams/{teamId}/members/{memberId}`
- 移交队长使用接口 `POST /api/teams/{teamId}/transfer`
- 重置邀请码使用接口 `POST /api/teams/{teamId}/invite-code/reset`

## 当前前端原则

- 保持 Nuxt UI 极简布局
- 队长和队员看到的操作保持明确分层
- 优先服务“创建队伍 -> 邀请队友 -> 报名比赛”的主流程
- 如果队伍已经被比赛锁定，优先直接展示锁定原因和关联比赛，而不是让用户靠接口报错理解规则

## 推荐使用方式

- 队长创建队伍后，优先直接复制邀请链接发给队友，而不是单独转发邀请码
- 队友打开邀请链接后，会自动填入邀请码，只需要登录并确认加入
- 如果邀请码已经发错对象或需要停用旧入口，队长可以先重置邀请码，再继续分发新的邀请链接
- 如果队长需要停用当前账号或退出队伍，先在队伍页把管理权移交给其他成员
- 队伍组建完成后，再统一前往比赛详情页完成报名，避免报名通过后再改成员导致锁队
- 如果当前就是从比赛详情页过来的，直接在队伍页完成创建或加入即可，系统会自动回跳，不需要手动再找比赛入口
- 如果队伍已经创建完成但还没报名任何比赛，页面会优先把“浏览比赛”作为下一步，而不是只停留在邀请码说明
